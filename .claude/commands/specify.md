세션 인수인계 블록을 읽고 이번 세션에서 처리 가능한 분량을 자동 선정한 뒤, 간결한 세션 플랜을 대화 내 텍스트로 출력하라.

전제: `/boot`로 컨텍스트가 이미 복원되었거나, 본 커맨드 실행 중 자체 복원을 포함한다. 인자 없음.

## 핵심 제약

- **Plan mode에 자발적으로 진입하지 않는다.** 플랜 파일을 생성하지 않는 것이 기본. 사용자가 이미 Plan mode에 있는 경우에만 플랜 파일을 쓸 수 있다.
- **handoff 블록 읽기·범위 확정 단계에서는 서브에이전트를 호출하지 않는다.** 해당 단계에서는 Grep/Glob/Read로 직접 수행. 범위 확정 후 실행 단계에서는 기존 가이드(탐색 3쿼리 이상이면 Explore 등)를 따른다.
- **2-3개 후보 비교 금지.** 최상위 1건을 자동 선정하고 단답 확인만 받는다.

## 절차

1. 참조할 handoff 파일 결정:
   - IDE 활성 파일이 `docs/session-reports/*-handoff.md` 패턴이면 그 파일
   - 아니면 `docs/session-reports/`에서 `ls -t`로 최신 파일
   - `CLAUDE_HANDOFF_START` ~ `CLAUDE_HANDOFF_END` 블록만 읽는다 (산문 전체 읽기 금지)
   - `/boot`로 이미 로드되었으면 재읽기 생략

2. 최상위 1건 자동 선정:
   - `next` 리스트 순서와 `unresolved` 우선순위(HIGH > MED > LOW) 교차 참조
   - 선행 의존이 미완인 항목은 건너뛴다
   - `traps`·`decisions` 블로커(모델 분화, 로컬/프로덕션 불일치, Claude 대필 금지 항목 등) 제외
   - 세션 1회(코드 변경 1-3파일 또는 조사 1건) 규모로 완결 가능한 것 선택
   - 민감 파일(`.env*`, 시크릿 등) 변경 포함 시 경고 1줄 추가

3. 단답 확인 1회:
   - 선정 결과를 한 문단으로 제시하고 "이 범위로 진행할까요?" 수준으로 묻는다
   - 복잡한 AskUserQuestion 분기 금지
   - 사용자가 거절하면 대안 1건만 제시

4. 승인 후 세션 플랜 텍스트 블록 출력 (파일 미생성):
   - **Context**: 왜 이 작업인가 (1-2문장)
   - **변경 대상**: 파일 경로 (마크다운 링크)
   - **재사용 유틸**: 기존 함수/파일:라인 (해당 시만)
   - **검증**: curl / pytest / playwright / 수동 확인 중 해당 항목만
   - **세션 외 이월**: 이번 세션에서 밀어낼 항목 (해당 시만)

5. 종료 메시지: "세션 범위 확정 — 실행해도 될까요?" 로 끝낸다. 실제 실행은 사용자의 다음 지시에서 시작.

## 주의

- handoff의 HIGH라도 프로덕션 기동이 필요한 항목은 기동 상태를 먼저 확인
- 저작 범위 블록, rotate 등 "Claude 대필 금지" 항목은 후보에서 자동 제외
