## 5. E-deep 머지 후 비교 방법

### 재측정 절차

```bash
source integrated_PARK/.env

# 동일 조건으로 100건 수집
curl -s "$BACKEND_HOST/api/v1/logs?type=queries&limit=100" \
  -H "X-API-Key: $API_SECRET_KEY" | python3 -m json.tool > after.json
```

### 비교 포인트

| 지표 | 측정 대상 | 예상 변화 |
|------|----------|----------|
| finance avg | Monte Carlo 비동기화 효과 | 이벤트 루프 블로킹 2-3s 제거 → 전체 파이프라인 latency 감소 |
| 전체 p90 | 타임아웃 핸들러 효과 | 직접적 latency 변화 없음 (에러 메시지 개선만) |
| 인메모리 세션 히트율 | LRU 교체 효과 | 프로덕션은 Cosmos DB 사용 → 인메모리 폴백에서만 의미 |

### 유의사항

- finance 에이전트 샘플 수(n=5)가 적어 통계적 유의성 확보 어려움
- 시간대·트래픽 패턴에 따른 Azure OpenAI 응답 변동 존재
- before/after 비교 시 동일 질문 세트로 재현 테스트 권장
