# 상권분석 에이전트 통합 테스트 목록

## Context
프론트엔드(MapView + ChatPanel)와 상권분석 에이전트(integrated_PARK / CHOI locationAgent_DB) 연동 과정에서 다수의 버그와 UX 문제가 확인됨. 본 문서는 문제를 체계적으로 재현·검증하기 위한 테스트 목록이다.

---

## 테스트 환경 전제
- 백엔드: 로컬 `integrated_PARK/api_server.py` 또는 CHOI `locationAgent_DB/api_server.py` 중 하나 구동
- 프론트엔드: `frontend/` npm run dev
- DB: OracleDB (`CHOI/locationAgent_DB/db/repository.py` 참조, 추후 Azure PostgreSQL 전환 예정)

## T2. 지도 클릭 → AI 분석 흐름 테스트

| # | 시나리오 | 기대 결과 | 확인 항목 |
|---|----------|-----------|-----------|
| T2-1 | 지도에서 행정동 폴리곤 클릭 후 DongPanel "🤖 AI" 버튼 클릭 → 채팅창에 "카페" 입력 | 해당 동 카페 분석 | 차트 제목에 동 코드(`__adm_XXXXXXXX`)가 아닌 **한국어 동이름** 표시 |
| T2-2 | T2-1 결과 차트 제목 형식 | `역삼1동 전체 카페 기준` 형태 | 코드 노출 없음 확인 |
| T2-3 | 지도 동 선택 후 채팅창에 업종 없이 문장 입력 ("이 지역 어때?") | 업종 요청 안내 | 자동 조합 오작동 없음 |
| T2-4 | 지도 동 선택 상태에서 전혀 다른 지역 명시 ("강남 한식 분석") | 강남 한식 분석 | mapContext 지역명 무시하고 명시 지역 우선 처리 |
| T2-5 | AI 분석 완료 후 지도 하이라이트 | 해당 동 폴리곤 하이라이트 + 자동 줌 | adm_codes 반환 → 지도 반영 여부 |
| T2-6 | T2-5에서 반환된 `adm_codes`가 빈 배열일 때 | 하이라이트 없음 (오류 아님) | 프론트엔드 오류 없음 |

## T3. 차트 생성 테스트

| # | 시나리오 | 기대 결과 | 확인 항목 |
|---|----------|-----------|-----------|
| T3-1 | 단일 지역 분석 시 차트 4개 반환 | 요일별·시간대별·연령대별·성별 차트 | base64 PNG 4개 렌더링 |
| T3-2 | 차트 제목(suptitle) 형식 | `{동이름} 전체 {업종} 기준` | 동 코드 미포함 |
| T3-3 | 요일별 차트 값 | 월평균 / 4 = 일단위 평균 | 이전 월평균 값과 비교해 1/4 수준 |
| T3-4 | 요일별 부제목 | `요일별 평균 매출(일단위)` | 텍스트 정확히 일치 |
| T3-5 | 주중/주말 폴백 차트 부제목 | `주중 / 주말 평균 매출(일단위)` | 시간대 데이터 없는 지역에서 확인 |
| T3-6 | 다중 지역 비교 시 차트 분리 | 월매출·점포당 평균·점포지표 3개 차트 분리 | 월매출과 점포당 평균이 별도 차트 |
| T3-7 | 비교 차트에서 스케일 차이 | 각 차트 독립 Y축 | 값이 작은 항목도 시각적으로 확인 가능 |
| T3-8 | 연령대·성별 데이터 없는 지역 | 해당 차트 "데이터 없음" 표시 또는 미생성 | 오류 없이 나머지 차트 정상 표시 |
