## 2. 구조 변경: 독립 앱 → APIRouter

### 원본 패턴 (backend/)

```python
# realEstateController.py
app = FastAPI(lifespan=lifespan)
app.add_middleware(CORSMiddleware, allow_origins=["*"], ...)

@app.get("/realestate/sangkwon")
async def getSangkwon(...):
    ...
```

각 컨트롤러가 **독립된 FastAPI 앱**이었습니다.
`uvicorn mapController:app --port=8681`처럼 **별도 포트**에서 각각 실행했습니다.

### 현행 패턴 (integrated_PARK/)

```python
# realestate_router.py
router = APIRouter()

@router.get("/realestate/sangkwon")
def getSangkwon(...):
    ...
```

`APIRouter`로 전환 후 `api_server.py`에서 **하나의 앱에 마운트**합니다.
CORS 설정, lifespan, 로깅 초기화는 모두 `api_server.py`가 담당합니다.

## 4. 핵심 변경: async def → def (이벤트 루프 블로킹 수정)

이것이 **가장 중요한 변경점**입니다.

### 문제 (원본 코드)

```python
# backend/realEstateController.py
@app.get("/realestate/sangkwon")
async def getSangkwon(...):           # ← async def
    row = skDAO.getSalesByCode(adm_cd)  # ← 동기 psycopg2 호출
    return ...
```

`async def` 핸들러 안에서 동기 psycopg2 DB 호출이 실행되면
uvicorn의 **이벤트 루프 전체가 블로킹**됩니다.
다른 모든 요청이 이 쿼리가 끝날 때까지 기다려야 합니다.

실제로 이 패턴이 원인이 되어 2026-04-03 04:28 UTC에
Azure Container App이 **무응답 상태**로 들어갔습니다.

### 수정 (현행 코드)

```python
# integrated_PARK/realestate_router.py
@router.get("/realestate/sangkwon")
def getSangkwon(...):                 # ← def (동기)
    row = skDAO.getSalesByCode(adm_cd)  # ← 동기 psycopg2 호출
    return ...
```

FastAPI는 **동기 함수(`def`)를 자동으로 스레드풀에서 실행**하므로
이벤트 루프를 블로킹하지 않습니다.

### 변경된 핸들러 목록 (realestate_router.py)

| 함수 | 원본 | 현행 | 이유 |
|---|---|---|---|
| `getSangkwon` | `async def` | `def` | 동기 DB 호출만 포함 |
| `getSangkwonBySvc` | `async def` | `def` | 동기 DB 호출만 포함 |
| `getSangkwonSvcByCat` | `async def` | `def` | 동기 DB 호출만 포함 |
| `getSangkwonStore` | `async def` | `def` | 동기 DB 호출만 포함 |
| `getSangkwonByInduty` | `async def` | `def` | 동기 DB 호출만 포함 |
| `getSangkwonQuarters` | `async def` | `def` | 동기 DB 호출만 포함 |
| `searchDong` | `async def` | `def` | 동기 DB 호출만 포함 |

`getLandValue`와 `getSeoulRtms`는 **진짜 async 작업**이므로 `async def` 유지.

map_data_router.py에서도 동일 수정:

| 함수 | 원본 | 현행 |
|---|---|---|
| `getDongDensity` | `async def` | `def` |

## 5. getSeoulRtms: 순차 → 병렬 실행

### 원본 (순차)

```python
# backend/realEstateController.py L82-84
seoul      = await rtmsDAO.fetch_by_emd_cd(adm_cd)   # 완료까지 대기
officetel  = molitDAO.fetch_officetel_rent(adm_cd)    # 순차 실행
commercial = molitDAO.fetch_commercial_trade(adm_cd)  # 순차 실행
```

세 데이터 소스가 **직렬**로 실행됩니다.
총 응답 시간 = A + B + C

### 현행 (병렬)

```python
# integrated_PARK/realestate_router.py L66-70
seoul, officetel, commercial = await asyncio.gather(
    rtmsDAO.fetch_by_emd_cd(adm_cd),                             # 비동기 API 호출
    asyncio.to_thread(molitDAO.fetch_officetel_rent, adm_cd),    # 동기 → 스레드
    asyncio.to_thread(molitDAO.fetch_commercial_trade, adm_cd),  # 동기 → 스레드
)
```

세 소스가 **병렬** 실행됩니다.
총 응답 시간 = max(A, B, C)
`asyncio.to_thread()`로 동기 psycopg2 호출도 이벤트 루프 밖 스레드에서 안전하게 실행합니다.

## 6. mapController → map_data_router 변경 요약

`mapController.py`의 변경은 구조 전환과 lifespan 제거가 주요 내용이며,
엔드포인트 로직 자체는 거의 동일합니다.

- `getDongDensity`: `async def` → `def` (블로킹 수정)
- lifespan(startup 로그): `api_server.py`로 이관
- CORS: `api_server.py`로 이관
- CSV_DIR: `BASE_DIR` 기준으로 동일하게 유지
- 쿼리 LIMIT: `getStoresByAdmCd`에 `LIMIT 1500` 추가 (P1 수정 — 커밋 `41c5b83`)
