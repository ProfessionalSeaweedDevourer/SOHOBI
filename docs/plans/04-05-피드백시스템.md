# SOHOBI 피드백 시스템 구현 플랜

## Context

SOHOBI 서비스에 사용자 피드백 수집 및 사용 추적 시스템을 단계별로 추가한다.
계층 2(인라인 피드백 위젯) → 계층 3(사용 추적·체크리스트·리포트) → 계층 3-D(로드맵 투표) 순서로 진행.
각 계층은 독립적인 세션에서 구현하여 컨텍스트 소모를 분산한다.

---

## 코드베이스 파악 결과 (사전 작업 완료)

### 프론트엔드
- **응답 렌더링 컴포넌트**: `frontend/src/components/ResponseCard.jsx` — 에이전트 응답 마크다운 렌더링 담당. 이 컴포넌트 하단에 `<InlineFeedback />` 삽입
- **세션 ID**: `UserChat.jsx` / `DevChat.jsx`에서 `useState`로 관리. SSE 이벤트 `domain_classified`에서 수신하여 `setSessionId()` 호출
- **HTTP 클라이언트**: `frontend/src/api.js` — 네이티브 `fetch` 사용, base URL `VITE_API_URL`, API Key `X-API-Key` 헤더. Axios는 패키지에만 존재, 실제 사용 안 함.
- **디자인 토큰**: `frontend/src/styles/theme.css` — CSS 변수 (`--brand-blue: #0891b2`, `--brand-teal: #14b8a6`, `--brand-orange: #f97316`, `--grade-a: #10b981`)
- **message_id**: 현재 없음 → 응답의 `request_id` 활용
- **에이전트 도메인 키**: 백엔드 실제 값은 `admin`, `finance`, `legal`, `location`, `chat`

### 백엔드
- **Cosmos DB 클라이언트**: `integrated_PARK/session_store.py` — `azure.cosmos.aio`, `DefaultAzureCredential`, 비동기 패턴. 신규 라우터도 동일 패턴 사용
- **기존 컨테이너**: `sessions` (partition: `/id`)
- **라우터 등록**: `api_server.py`에서 `app.include_router(router)` 패턴 (Line 56-59)
- **기존 라우터 파일들**: `map_router.py`, `map_data_router.py`, `realestate_router.py`

### 피드백 데이터 활용 계획

- **저장**: Cosmos DB `feedback` 신규 컨테이너 (파티션키: `/agent_type`)
- **세션 1에서는 저장만** — 조회 API 미구현
- **추후 관리자 대시보드** 구현 시 `GET /api/feedback/summary`로 집계·시각화
- **세션 4(리포트)** 에서 `feedback_summary` (긍정률, top 태그) 사용자에게 노출

### 스타일링
- CSS 변수 사용: `--brand-blue`, `--brand-teal` 등 기존 테마 토큰
- 태그 칩: flex-wrap, 선택 시 `--brand-blue` 배경 + 흰색 텍스트
- 전체 영역: `border-top: 1px solid` + 연한 회색 구분선
- 태그 영역 150ms ease 전환
