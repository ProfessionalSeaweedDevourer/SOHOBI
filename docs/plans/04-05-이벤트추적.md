## 상세 구현

### 1. `frontend/src/utils/trackEvent.js`

`useFeedbackSubmit.js`의 fetch 패턴 참고. BASE_URL과 X-API-Key 헤더 방식 동일.

```js
const BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:8000';
const _HEADERS = {
  'Content-Type': 'application/json',
  ...(import.meta.env.VITE_API_KEY ? { 'X-API-Key': import.meta.env.VITE_API_KEY } : {}),
};

export async function trackEvent(eventName, payload = {}) {
  try {
    await fetch(`${BASE_URL}/api/events`, {
      method: 'POST',
      headers: _HEADERS,
      body: JSON.stringify({
        event_name: eventName,
        ...payload,
        timestamp: new Date().toISOString(),
      }),
    });
  } catch {
    // 실패 무시 — UX 영향 없음
  }
}
```

### 2. `integrated_PARK/event_router.py`

`feedback_router.py`와 동일한 싱글턴 + 인메모리 폴백 패턴.
- 컨테이너: `usage_events`
- 파티션 키: `/session_id`
- 저장 방식: `create_item()`

스키마:
```python
class EventRequest(BaseModel):
    event_name: str
    session_id: str | None = None
    agent_type: str | None = None
    message_id: str | None = None
    page: str | None = None
    timestamp: str | None = None
```

### 3. `integrated_PARK/api_server.py`

- `from event_router import router as event_router`
- `app.include_router(event_router)`

### 4. 이벤트 삽입

**`agent_query`** — `UserChat.jsx` `handleSubmit` 첫 줄 (streamQuery 호출 전):
```js
trackEvent('agent_query', { session_id: sessionId, page: 'user_chat' });
```
sessionId는 이 시점에 null일 수 있음 — 그대로 전달(백엔드에서 null 허용).

**`agent_response_view`** — `ResponseCard.jsx`에 useEffect 추가:
```js
useEffect(() => {
  if (status !== 'error') {
    trackEvent('agent_response_view', {
      session_id: sessionId,
      message_id: messageId,
      agent_type: domain,
    });
  }
}, []);
```

**`feature_discovery`** — 각 페이지 useEffect 마운트:

| 파일 | page 값 |
|------|---------|
| `Landing.jsx` | `'landing'` |
| `Home.jsx` | `'home'` |
| `Features.jsx` | `'features'` |
| `MapPage.jsx` | `'map'` |
| `UserChat.jsx` | `'user_chat'` |

```js
useEffect(() => {
  trackEvent('feature_discovery', { page: 'landing' });
}, []);
```

## 검증

1. 백엔드 로컬 실행: `cd integrated_PARK && .venv/bin/python3 api_server.py`
2. 이벤트 전송 테스트:
   ```bash
   curl -s -X POST http://localhost:8000/api/events \
     -H "Content-Type: application/json" \
     -d '{"event_name":"agent_query","session_id":"test","page":"user_chat"}'
   ```
   → `{"status": "ok", "id": "..."}` 응답 확인
3. 프론트엔드에서 페이지 진입 시 브라우저 Network 탭에서 `/api/events` 요청 확인
4. 에러 발생 시 콘솔에 아무 출력도 없어야 함 (실패 무시)
