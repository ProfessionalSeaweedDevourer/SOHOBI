## 3. 핵심 기능 & 사용자 여정

**권장 슬라이드: 3~4장**

> 기능 나열이 아닌 **사용자가 실제로 겪는 여정** 순서로 구성합니다.

### 여정 1: "카페 창업하고 싶은데 뭐부터 해야 하나요?"

- **Chat 에이전트**가 창업 절차를 개괄적으로 안내
- 사용자의 의도를 파악해 적절한 전문 에이전트로 안내
- 창업 **체크리스트** (8개 항목: 업종 결정, 입지 선정, 초기 자금, 사업자등록, 영업허가, 인력채용, 수익성검토, 임대차계약)가 화면 옆에 자동 표시

### 여정 2: "강남 vs 홍대, 어디가 좋을까요?"

- **Location 에이전트 (상권분석)**가 서울 2024년 4분기 실제 상권 데이터로 비교 분석
- 제공 정보: 월 매출, 시간대별 매출 분포, 고객 연령대/성별, 유사 상권 추천
- **인터랙티브 지도** (서울 행정동 단위)에서 시각적 탐색 가능
- 지적도, 공시지가, 랜드마크, 학교, 축제 등 레이어 오버레이

### 여정 3: "월 매출 3000만원, 임대료 500만원이면 수익이 나나요?"

- **Finance 에이전트 (재무 시뮬레이션)**가 Monte Carlo **10,000회** 시뮬레이션 실행
- 결과: 확률 분포 히스토그램 차트, 손실 확률, 안전 마진, P5/P20/P95 백분위
- 손익분기점(BEP), 투자금 회수 시나리오 자동 계산
- 누락 파라미터는 해당 지역/업종 평균값으로 자동 보정

### 여정 4: "임대차 계약서에서 주의할 조항이 뭐예요?"

- **Legal 에이전트 (법무)**가 Azure AI Search 기반 RAG로 한국 법령 DB 검색
- **구체적 법률명 + 조문 번호** 인용 (예: 상가건물 임대차보호법 제10조)
- 3줄 면책 고지 자동 포함 (AI 법률 자문의 한계 고지)

### 여정 5: "영업신고는 어떻게 하나요? 받을 수 있는 지원금은?"

- **Admin 에이전트 (행정)**가 5단계 행정 절차를 순서대로 안내:
  1. 사업자등록 → 2. 위생교육 → 3. 영업신고 → 4. 보건증 → 5. 소방
- **5,600+ 정부 지원 프로그램** 하이브리드 검색 (벡터 + 키워드)으로 수혜 가능한 프로그램 자동 매칭
  - 정책자금, 청년 창업 패키지, 노란우산공제, 지역 신보 보증, 고용지원금 등
- **PDF 자동 생성**: 식품영업신고서 양식을 대화형으로 수집 → 제출용 PDF 출력

### 전 과정에서: 창업 체크리스트 자동 추적

- 에이전트 답변 내 키워드를 기반으로 체크리스트 8개 항목이 **자동 체크**
- 진행률 바로 시각적 완성도 확인
- 미완료 항목에 대한 추천 질문 자동 생성

### 슬라이드 시각 제안

- 유저 저니 플로우 다이어그램: 왼쪽에 사용자 질문 말풍선, 오른쪽에 에이전트 답변 + 시각화 미리보기
- 각 여정을 카드 형태로 구성, 에이전트별 브랜드 컬러 구분
- Monte Carlo 히스토그램 차트 예시 이미지 (손실 구간=빨강, 위험 구간=노랑, 수익 구간=틸)

## 4. 시스템 아키텍처 — PM 관점

**권장 슬라이드: 2장**

### 핵심 처리 흐름

```
사용자 자연어 질문
    ↓
[2단계 도메인 라우터]
  - 1단계: 키워드 매칭 (2개 이상 일치 시 즉시 분류, 신뢰도 0.85)
  - 2단계: LLM 분류 (키워드 불일치 시 GPT-4o가 도메인 판별)
    ↓
[전문 에이전트 처리]  ←→  [전문 플러그인: 법령 RAG, 상권 DB, Monte Carlo 등]
    ↓
[Sign-off 품질 검증]
  - 도메인별 루브릭 기반 A/B/C 등급
  - Grade C → 자동 재시도 (최대 3회, 피드백 포함)
    ↓
검증된 최종 답변 (텍스트 + 차트 + PDF) → SSE 스트리밍으로 실시간 전달
```

### 3개 레이어

| 레이어 | 기술 | 역할 |
|--------|------|------|
| **프론트엔드** | React 19 + Vite 7, Tailwind CSS | 채팅 UI, 인터랙티브 지도, 차트, 체크리스트 |
| **오케스트레이션** | Microsoft Semantic Kernel | 도메인 라우팅 → 에이전트 선택 → Sign-off 품질 루프 |
| **백엔드 서비스** | FastAPI + Azure 서비스 군 | AI Foundry (GPT-4o), AI Search (RAG), PostgreSQL (상권), Cosmos DB (세션), Blob Storage (로그) |

### Sign-off 품질 검증 시스템

| 등급 | 의미 | 처리 |
|------|------|------|
| **Grade A** | 모든 루브릭 기준 충족 | 즉시 승인, 사용자에게 전달 |
| **Grade B** | 경미한 미흡 | 경고와 함께 승인 |
| **Grade C** | 핵심 기준 미달 | 자동 재생성 (피드백 포함, 최대 3회) |

각 에이전트 도메인별로 고유 루브릭 코드가 존재 (공통 C1~C5 + 도메인별 추가 코드).

### 슬라이드 시각 제안

- 슬라이드 1: 전체 처리 흐름도 (위 텍스트를 시각 다이어그램으로, Sign-off 루프를 강조)
- 슬라이드 2: 3-레이어 아키텍처 + Azure 서비스 로고 배치

## 5. 기술 차별점

**권장 슬라이드: 2장**

### 1. Sign-off 품질 보증 — "AI가 만든 답변을 AI가 검증"

- 단순 생성이 아닌, **도메인별 루브릭 기반 자동 품질 검증**
- Grade C 시 원인 피드백을 포함해 재생성 → 최종 답변 품질 보장
- 실측 결과: 전체 에이전트 **84~100% Grade A** 달성

### 2. 2단계 도메인 라우팅 — "정확한 전문가에게 즉시 연결"

- 1단계 키워드 매칭으로 불필요한 LLM 호출 제거 → 비용/속도 최적화
- 2단계 LLM 분류로 모호한 질문도 정확히 분류
- 5개 도메인: admin, finance, legal, location, chat

### 3. 하이브리드 RAG — "법령 조문 번호까지 정확히 인용"

- Azure AI Search: 벡터 검색 + 키워드 검색 하이브리드
- 법률 에이전트는 반드시 **법률명 + 조문 번호**를 인용하도록 루브릭 강제
- 정부 지원 프로그램 5,600+건도 하이브리드 검색으로 정밀 매칭

### 4. Monte Carlo 재무 시뮬레이션 — "감이 아닌 확률로 판단"

- 10,000회 반복 시뮬레이션으로 **확률 분포** 제공 (단일 추정치 X)
- P5/P20/P95 백분위, 손실 확률, 안전 마진, 투자금 회수 시나리오
- ThreadPoolExecutor 비동기 처리로 이벤트 루프 블로킹 방지

### 5. Microsoft Azure 네이티브 스택

- AI Foundry (GPT-4o), Container Apps, Static Web Apps, PostgreSQL, Cosmos DB, AI Search, Blob Storage, DNS
- Azure 생태계 내에서 완결되는 풀스택 아키텍처

### 슬라이드 시각 제안

- 각 차별점을 아이콘 + 한 줄 요약 + 핵심 수치로 카드 형태 배치
- Sign-off 루프를 순환 다이어그램으로 시각화

| **정부 지원 프로그램** | 5,600+건 | Azure AI Search | 업종/지역/자격 기반 수혜 가능 프로그램 하이브리드 검색 |
| **서울 상권 데이터** | 2024년 Q4 | Azure PostgreSQL | 행정동별 매출, 시간대, 성별, 연령대, 업종별 분석 |
| **한국 법령 DB** | 생활법령정보 | Azure AI Search (RAG) | 법률명 + 조문 인용 기반 법률 상담 |
| **행정 절차 KB** | 5개 핵심 절차 | 검증된 지식 베이스 | 사업자등록~소방까지 단계별 안내 |
| **재무 벤치마크** | 지역/업종 평균 | 내장 데이터 | 누락 파라미터 자동 보정용 기본값 |

### AI 파이프라인 흐름

```
[입력]                    [처리]                         [출력]
자연어 질문          →   도메인 분류                  →   구조화된 한국어 답변
+ 세션 컨텍스트      →   에이전트별 플러그인 호출     →   차트/시각화
                     →   LLM 합성 (GPT-4o)           →   PDF 문서
                     →   Sign-off 품질 검증           →   법령 인용
                     →   (Grade C 시 재시도)          →   정부지원 매칭 결과
```

### 핵심 AI 모델

- **LLM**: Azure AI Foundry — GPT-4o (에이전트 응답 생성 + Sign-off 검증)

- **시뮬레이션**: NumPy + Matplotlib — Monte Carlo 10,000회 + 히스토그램
