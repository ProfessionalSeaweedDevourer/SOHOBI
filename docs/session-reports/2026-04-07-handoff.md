## 이번 세션에서 해결한 문제

### 전체 에이전트 404 → 해결

**원인 1 (코드, PR #186으로 수정)**:
- `model_config.py`의 `API_VERSIONS` 딕셔너리가 **모델 버전 날짜**(예: `gpt-5.4-mini` → `2026-03-17`)를 Azure OpenAI REST API 버전으로 잘못 사용
- `?api-version=2026-03-17`은 유효하지 않은 API 버전 → 404
- 해결: `_api_version()` 함수 및 `API_VERSIONS` import 제거, `AZURE_OPENAI_API_VERSION` 환경변수(`2024-05-01-preview`) 단일 사용

**원인 2 (인프라, az CLI로 수정)**:
- 이전 세션에서 `AZURE_OPENAI_ENDPOINT`를 `cognitiveservices.azure.com`으로 변경 → gpt-5.4 계열 미접근
- 최종 확인: `openai.azure.com/openai/deployments/gpt-5.4/` + `api-version=2024-05-01-preview` 정상 작동
- 해결: `AZURE_OPENAI_ENDPOINT`, `AZURE_SIGNOFF_ENDPOINT` → `https://ejp-9638-resource.openai.azure.com/` 복원

### 검증된 올바른 접근 방법

```
엔드포인트: https://ejp-9638-resource.openai.azure.com/
경로:        /openai/deployments/{deployment}/chat/completions
파라미터:    ?api-version=2024-05-01-preview
파라미터:    max_completion_tokens (max_tokens 사용 불가)
인증:        DefaultAzureCredential (managed identity)
```

## 다음 세션 작업

### 1. 응답 속도 최적화 (선택)

법률 40초는 체감 지연 큼. 필요 시:

- SSE 스트리밍 (`run_stream()`) 프론트엔드 활용도 점검

### 2. 동적 모델 선택 (장기)

1. `kernel_setup.py` — `get_kernel(tier: str = "free")` 시그니처 추가, `MODEL_CONFIG[tier][domain]` 배포명 선택
2. `orchestrator.py` — `run()`/`run_stream()`에 `user_tier: str = "free"` 추가
3. `api_server.py` — `QueryRequest`에 `user_tier: Literal["free","premium"] = "free"` 추가
4. `finance_agent.py` — `finance_param` / `finance_explain` deployment 분리

### 3. model_config.py 정리 (기술 부채)
