**증상:** 라우팅은 정상(키워드 매칭), 에이전트 응답은 미분류로 기록됨

## 원인 분석

### 원인 1 (주요) — kernel_setup.py base_url 변경

PR #183의 `kernel_setup.py` 수정이 문제:

| 구분 | 이전 (정상) | PR #183 (오류) |
|------|------------|----------------|
| 파라미터 | `endpoint=endpoint` | `base_url=base_url` |
| SDK 동작 | `{endpoint}/openai/deployments/{name}/chat/completions` | `{base_url}/chat/completions` |
| 결과 | 정상 | 404 — classic Azure OAI에 `/chat/completions` 경로 없음 |

**→ 이번 세션에서 핫픽스 적용 완료**: `base_url=` → `endpoint=`, api_version 기본값 `2024-05-01-preview`로 복원

### 원인 2 (잠재) — 도메인별 배포 변수 미설정

`.env`에 `AZURE_ADMIN_DEPLOYMENT` 등 도메인별 변수가 없음 → `AZURE_DEPLOYMENT_NAME=gpt-4.1-mini` fallback.
신규 AI Foundry 엔드포인트에 `gpt-4.1-mini`가 없으면 추가 404 발생 가능.

## 다음 세션이 해야 할 일

### 즉시 (서비스 복구)

**Step 1 — .env 업데이트** (사용자 직접):

```bash
# 엔드포인트: cognitiveservices.azure.com 형식 (endpoint= 파라미터와 호환)
AZURE_OPENAI_ENDPOINT=https://ejp-9638-resource.cognitiveservices.azure.com

# 기본 배포명 갱신
AZURE_DEPLOYMENT_NAME=gpt-5.4

# 도메인별 배포명 (현재 모두 미설정 — gpt-4.1-mini fallback 발생)
AZURE_ADMIN_DEPLOYMENT=gpt-5.4
AZURE_FINANCE_DEPLOYMENT=gpt-5.4
AZURE_LEGAL_DEPLOYMENT=gpt-5.4
AZURE_LOCATION_DEPLOYMENT=gpt-5.4
AZURE_CHAT_DEPLOYMENT=gpt-5.4-mini
AZURE_ROUTER_DEPLOYMENT=gpt-5.4-mini
AZURE_SIGNOFF_DEPLOYMENT=gpt-5.4

# API 버전
AZURE_OPENAI_API_VERSION=2024-05-01-preview

# API 키 없음 (AD 토큰 유지)
AZURE_OPENAI_API_KEY=
```

**Step 2 — 검증:**

```bash
source integrated_PARK/.env
curl -s -X POST "$BACKEND_HOST/api/v1/query" \
  -H "Content-Type: application/json" \
  -d '{"question": "강남 카페 상권 어때"}' | python3 -m json.tool
```

- `domain: "location"` 으로 정상 분류되는지 확인
- 재무 질문 (`월매출 2000만, 원가 600만이면 수익은?`)으로 동일 확인

### 이후 (안정화 후)

1. **model_config.py 활성화** — `cognitiveservices.azure.com`에서 gpt-5.4/gpt-5.4-mini 배포 확인 후
2. **kernel_setup.py tier 분기** — `user_tier` 파라미터 추가, `MODEL_CONFIG[tier][domain]`으로 배포명 선택
3. **orchestrator.py / api_server.py** — `user_tier` 전달 경로 추가

### 도메인 라우터가 정상인 이유

`domain_router.py`는 키워드 매칭 단계(`_keyword_classify`)에서 성공하면 LLM을 호출하지 않음.
상권·재무 키워드 매칭 성공 → LLM 없이 라우팅 → 에이전트 실행 단계에서 404 발생하는 구조.
