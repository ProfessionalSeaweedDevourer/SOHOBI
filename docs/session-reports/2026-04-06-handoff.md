### 작업 1 — LogViewer 세션 ID 표시 및 세션 기반 조회

**배경**: 현재 LogViewer에서 각 로그 항목의 `session_id`가 노출되지 않는다. 비회원 어뷰징 탐지나 특정 대화 흐름 추적 시 session_id 단위 조회가 필요하다. 또한 로그인 회원의 경우 session_id를 클릭하면 `/api/my/sessions/{session_id}/history`로 실제 대화 내역을 조회할 수 있다.

#### 1-1. `frontend/src/components/LogTable.jsx` — session_id 표시

`EntryDetail` 메타 영역에 session_id 추가 (전체 표시, 복사 버튼 제공):

```jsx
{entry.session_id && (
  <span
    className="text-muted-foreground font-mono text-xs truncate max-w-[160px] cursor-pointer hover:text-foreground"
    title={entry.session_id}
    onClick={() => navigator.clipboard.writeText(entry.session_id)}
  >
    🔗 {entry.session_id.slice(0, 8)}…
  </span>
)}
```

목록 항목(버튼)에는 전체 session_id 대신 앞 8자만 표시(공간 제약).

#### 1-2. `frontend/src/pages/LogViewer.jsx` — 세션 기반 필터

현재 사용자 드롭다운 필터 옆에 session_id 검색 input 추가:

```jsx
const [sessionFilter, setSessionFilter] = useState("");

// 검색 UI
<input
  type="text"
  placeholder="세션 ID 검색..."
  value={sessionFilter}
  onChange={(e) => setSessionFilter(e.target.value)}
  className="text-xs rounded-lg px-2 py-1 border border-[var(--border)] bg-[var(--card)] text-foreground w-40"
/>
```

필터링은 **프론트엔드 레벨**에서 처리 (백엔드 파라미터 추가 없이):
```javascript
const displayEntries = entries.filter((e) =>
  !sessionFilter || e.session_id?.includes(sessionFilter)
);
```

> session_id 필터는 자유 검색(포함 여부)으로, 사용자 필터와 AND 조건으로 결합.

#### 1-3. `integrated_PARK/api_server.py` — `/api/v1/logs` session_id 필터 (선택)

필요 시 백엔드에도 `session_id: str = ""` 파라미터 추가 (프론트 필터만으로 충분하면 생략 가능):

```python
@app.get("/api/v1/logs", dependencies=[Depends(verify_api_key)])
async def get_logs(type: str = "queries", limit: int = 50, user_id: str = "", session_id: str = ""):
    ...
    if session_id:
        enriched = [e for e in enriched if session_id in e.get("session_id", "")]
```

LogViewer에서 특정 session_id 일부를 입력했을 때 해당 항목만 필터링되는지 확인.

### 작업 2 — 비회원 IP 수집 (기술)

**목적**: 비회원 응답자를 IP로도 식별하여 어뷰징 탐지 및 보안 모니터링 강화.
**법적 근거**: 개인정보 보호법 제15조 제1항 제6호 정당한 이익. 처리방침 제3조에 이미 "IP 주소" 자동 수집이 명시되어 있어 추가 동의 절차 불필요 (단, 아래 작업 3에서 구체화 필요).

#### 2-1. `integrated_PARK/logger.py`

`log_query()` 및 `log_error()`에 `client_ip: str = ""` 파라미터 추가:

```python
def log_query(*, ..., user_id: str = "", client_ip: str = "", ...):
    record = {
        ...,
        "user_id":   user_id,
        "client_ip": client_ip,
        ...
    }
```

#### 2-2. `integrated_PARK/api_server.py`

IP 추출 헬퍼 추가 + 핸들러 2곳에 전달:

```python
def _get_client_ip(request: Request) -> str:
    forwarded = request.headers.get("X-Forwarded-For", "")
    if forwarded:
        return forwarded.split(",")[0].strip()
    return request.client.host if request.client else ""
```

> **주의**: `/api/v1/query`는 `async def query(req: QueryRequest)` 형태. `request: Request`를 두 번째 파라미터로 추가해야 함. `/api/v1/stream`도 동일.

```python
async def query(req: QueryRequest, request: Request):
    ...
    log_query(..., client_ip=_get_client_ip(request))
```

#### 2-3. `frontend/src/components/LogTable.jsx` — IP 표시

`EntryDetail` 메타 영역에 추가 (관리자 전용 뷰이므로 노출 무방):

```jsx
{entry.client_ip && (
  <span className="text-muted-foreground font-mono" title="접속 IP">
    🌐 {entry.client_ip}
  </span>
)}
```

#### 검증

