### 1. Azure Container Apps 배포 파이프라인 — `51799f8` (#29)

Railway → Azure Container Apps 전환. 내부망 기반 Azure 리소스 연동으로 전환.

- `integrated_PARK/Dockerfile` — 멀티스테이지 빌드 (`python:3.12-slim-bookworm`, wkhtmltopdf 포함)
- `.github/workflows/deploy-backend.yml` — ACR 빌드/푸시 → Container App 자동 배포 (OIDC)
- Azure 리소스 구성: Container Registry (`sohobi9638acr`), Container Apps 환경 (`sohobi-env`), Container App (`sohobi-backend`)
- OIDC 서비스 주체 + GitHub 시크릿 6개 등록

### 2. Cosmos DB Stateless 세션 스토어 — `451e28a` (#24)

인메모리 세션에서 Azure Cosmos DB 기반 Stateless 세션으로 전환.

- `integrated_PARK/session_store.py` — Cosmos DB 읽기/쓰기 DAO 구현
- Container App 재시작 시에도 세션 유지

### 3. Azure Blob Storage 로그 영구 저장 — `40b3817` (#31)

Container App은 에페머럴(임시)이라 재시작 시 로컬 로그 소멸. Blob Storage에 append 방식으로 영구 저장.

- `BLOB_LOGS_ACCOUNT` 환경변수 설정 시 Blob 사용, 미설정 시 로컬 폴백
- Azure Storage Account `sohobi9638logs` / 컨테이너 `sohobi-logs` 구성

### 4. SSE 실시간 스트리밍 — `7e5bc7d` (#23)

에이전트 처리 진행 상황을 Server-Sent Events로 실시간 전송.

- `event: agent_start`, `event: signoff_start`, `event: signoff_result`, `event: complete` 이벤트 스트림
- 프론트엔드: 진행 상황 표시 컴포넌트 연동

### 5. Signoff 에이전트 gpt-5.4-pro + Responses API 전환 — `54ce0c2`

- Responses API 기반 추론 모델 signoff 구현 (이후 Chat Completions로 재전환 예정)

### 6. SWA 라우팅 설정 및 VITE_API_URL 빌드 주입 — `1c4748a`, `1fb11e7` (#26)
